## 1. Executive Summary  
  
The mobile Sudoku market is mature, crowded, and segmented. Over 200 apps compete for the same search terms. Yet the competitive analysis reveals a clear and unoccupied position: **no app combines teaching-quality hints, calm ad-free design, structured progression, replayable solve records, transparent scoring, and fair ranked competition on both platforms.**  
  
This document synthesizes competitive research across seven dimensions — gameplay, hints, scoring, replay, game history, leaderboards, and app store presence — to demonstrate that Orbace Sudoku's Su-Pu (数谱) framework represents a category-defining innovation that no competitor is positioned to replicate quickly.  
  
**Key finding**: The Su-Pu concept — a recorded solve that is simultaneously a learning record, a performance record, a collectible artifact, and a shareable achievement — has no equivalent in any Sudoku app. The closest analog exists in a different category entirely (Chess.com's game records). This creates a first-mover window for Orbace to define the standard for recorded Sudoku solves.

## 2. Market Structure & Segmentation  
  
### 2.1 Category Overview  
  
The mobile Sudoku market divides into four distinct segments:  
  
| Segment | Market Share | Monetization | User Profile | Examples |  
|---------|-------------|--------------|--------------|----------|  
| **Mass-Market Free** | ~60% | Heavy ads, IAP to remove | Casual, ad-tolerant, high churn | Sudoku.com (Easybrain), Sudoku - Classic Puzzle |  
| **Premium Indie** | ~5% | Paid upfront ($3-7) | Design-sensitive, puzzle enthusiasts | Good Sudoku (Zach Gage) |  
| **Utility/Power User** | ~15% | Free/cheap, some ads | Feature-maximalist, variant lovers | Andoku 3, Sudoku 10000 |  
| **Casual Game** | ~20% | IAP, lives, boosters | Mobile gamers, progression-seekers | Sudoku Quest, Sudoku Adventure |  
  
### 2.2 Market Dynamics  
  
| Dynamic | Implication for Orbace |  
|---------|----------------------|  
| Category leader (Sudoku.com) has 100M+ downloads but 30-40% of critical reviews complain about ads | "No ads during play" is a powerful acquisition message |  
| Best teaching app (Good Sudoku) is iOS-only and paid-upfront | Cross-platform freemium with teaching creates accessibility |  
| No app has meaningful replay or solve recording | First-mover advantage in Su-Pu concept |  
| Leaderboards are either absent or speed-dominant | Accuracy-weighted ranking with integrity verification is unique |  
| App store visuals are homogenous (blue grids, feature lists) | Cultural design layer (seal icon, Ink Wash themes) creates visual distinction |  
  
### 2.3 The #1 User Complaint Across the Category  
  
Across all ad-supported Sudoku apps, the dominant negative review sentiment is:  
  
> "Too many ads." / "Ads after every puzzle." / "Full screen video ads."  
  
**No major competitor leads their store presence with an ad-free promise.** This is Orbace's primary acquisition wedge.

## 3. Competitor Profiles & SWOT Analysis  
  
### 3.1 Sudoku.com (Easybrain)  
  
**The category leader. Over 100 million downloads. Sets expectations for the mass market.**  
  
| Dimension | Assessment |  
|-----------|------------|  
| **Puzzle Quality** | Good. Solver-validated. Consistent difficulty within broad buckets. Thousands of puzzles. |  
| **Hints** | Reveal a random cell. No explanation. No technique names. No tiered escalation. |  
| **Board UX** | Clean, responsive, good highlighting. Notes, undo, auto-check. |  
| **Scoring** | Not a focus. Time and difficulty tracked but no composite score. No breakdown. |  
| **Replay** | None. No solve recording. Puzzle completed = puzzle gone. |  
| **Game History** | None. Aggregate stats only (games played, win rate, streaks, best times). No per-puzzle history. |  
| **Leaderboards** | None. |  
| **Progression** | Daily Challenges with seasonal themes. No structured level packs. No awards. |  
| **Monetization** | Aggressive ads: full-screen video after every puzzle, banner during play. IAP to remove: $4.99/week or $29.99/year. |  
| **Platform** | iOS + Android. |  
| **App Store** | 4.6★ (iOS, 300K+ ratings). Professional but generic screenshots. Blue grid icon. |  
  
**SWOT Analysis:**  
  
| Strengths | Weaknesses |  
|-----------|------------|  
| Massive brand recognition and install base | Ads are the #1 user complaint |  
| Reliable puzzle generation at scale | Hints don't teach — reveal only |  
| Strong daily challenge engagement | No replay, no game history, no progression |  
| Cross-platform | Timer always visible, creates background anxiety |  
| Seasonal content keeps experience fresh | Score and performance tracking is shallow |  
  
| Opportunities | Threats |  
|---------------|---------|  
| Their ad-frustrated users are Orbace's acquisition target | Could add teaching hints if they chose to invest |  
| Could improve hints and retain users longer | Deep pockets for feature development |  
| — | Market dominance creates high barrier for discovery |

### 3.2 Good Sudoku (Zach Gage)  
  
**The quality ceiling. Apple Design Award finalist. Best-in-class teaching and board feel. iOS only.**  
  
| Dimension | Assessment |  
|-----------|------------|  
| **Puzzle Quality** | Excellent. Hand-tuned difficulty. Consistent solving experience. Smaller curated library (70,000+). |  
| **Hints** | **Best-in-class.** Interactive tutorial teaches techniques. Hints show "next logical step" with technique names (Naked Single, Hidden Pair). Animated cell highlights. AI-powered hint support. |  
| **Board UX** | Beautiful. Typography-forward. Satisfying completion animation. Excellent highlighting. Smart note auto-removal. Timer off by default. |  
| **Scoring** | None. Deliberate design choice against competition. |  
| **Replay** | None. Can view static completed board. No playback. No move history. |  
| **Game History** | "Previous Puzzles" section: recent puzzles with date and time. Reverse-chronological list only. No filtering. No multiple attempts. No organization. |  
| **Leaderboards** | None. Philosophical choice. |  
| **Progression** | No level packs. Random puzzles by difficulty. "Daily" section with curated puzzles. No streak tracking. |  
| **Monetization** | $6.99 one-time purchase. No ads, no IAP. |  
| **Platform** | iOS only. |  
| **App Store** | 4.8★ (iOS, 8K+ ratings). Design-forward screenshots. Apple Design Award badge. |  
  
**SWOT Analysis:**  
  
| Strengths | Weaknesses |  
|-----------|------------|  
| Best teaching system in the category | iOS only — cedes entire Android market |  
| Premium design and UX | Paid upfront — limits audience reach |  
| Apple Design Award credibility | No progression system — random play only |  
| Technique-aware difficulty and hints | No competitive features |  
| Respected in puzzle community | No replay, no game history depth |  
  
| Opportunities | Threats |  
|---------------|---------|  
| — | Could launch on Android (most significant competitive threat) |  
| — | Could add progression/replay if they change philosophy |  
| — | Their hint system is the benchmark Orbace must match |

### 3.4 Andoku 3  
  
**The power user's choice. Feature-maximalist. Variant-heavy. Android only.**  
  
| Dimension | Assessment |  
|-----------|------------|  
| **Puzzle Quality** | Good generation. Supports many variants (Killer, X-Sudoku, Squiggly, etc.). |  
| **Hints** | None. Reveals cells. No teaching. Assumes player knows Sudoku. |  
| **Board UX** | Functional, not beautiful. Dense menus. Many settings. Multiple input modes. |  
| **Scoring** | Points based on difficulty + time. Customizable but opaque defaults. |  
| **Replay** | None. Move history for undo only. No playback. |  
| **Game History** | **Best data model in category.** Saves each play separately. Supports multiple concurrent games. Can export/import data. Organized by variant type and difficulty. But: UX is developer-oriented. History is buried in menus. |  
| **Leaderboards** | None. |  
| **Progression** | No structured progression. Difficulty selector + endless play. |  
| **Monetization** | Free with ads. One IAP to remove. |  
| **Platform** | Android only. |  
| **App Store** | 4.7★ (Android, 50K+ ratings). Technical screenshots. Appeals to power users, intimidates casuals. |  
  
**SWOT Analysis:**  
  
| Strengths | Weaknesses |  
|-----------|------------|  
| Best variant support in category | Android only |  
| Strong data model (multiple attempts, export) | UX is developer-oriented, not player-friendly |  
| Multiple concurrent games supported | No teaching features |  
| Data portability (export/import) | No replay despite having the data |  
| Power user trust | Visually dated |  
  
| Opportunities | Threats |  
|---------------|---------|  
| Their data model proves Su-Pu is technically feasible | Owns the "variant Sudoku" segment |  
| — | Orbace should not compete on variant quantity |  
| — | Their export feature is the benchmark for data portability |

### 3.5 Cracking the Cryptic  
  
**Niche but influential. YouTube-famous for variant Sudoku. Community-driven.**  
  
| Dimension | Assessment |  
|-----------|------------|  
| **Puzzle Quality** | **Highest quality.** Hand-crafted by celebrity constructors. Very hard. Not generated. |  
| **Hints** | Video walkthroughs for every puzzle. Community-driven learning. Not in-app hint system. |  
| **Board UX** | Clean but utilitarian. Not design-led. Web-based + app hybrid. |  
| **Scoring** | None. Community tracks solve times informally. |  
| **Replay** | Browser-based video export (not mobile-native). Community shares solves on YouTube. |  
| **Game History** | Pack completion tracking. Implicit in which packs are owned/completed. No individual puzzle history. |  
| **Leaderboards** | Informal. Community-maintained solve times in comments/Discord. Not in-app. |  
| **Progression** | Puzzle packs. Themed collections. Clear pack ownership. |  
| **Monetization** | Free base app. Paid puzzle packs ($2-5 each). |  
| **Platform** | Web + iOS + Android (limited). |  
| **App Store** | Niche presence. Community-driven discovery. |  
  
**SWOT Analysis:**  
  
| Strengths | Weaknesses |  
|-----------|------------|  
| Highest puzzle quality (hand-crafted) | Not scalable — hand-crafted only |  
| Strong community and YouTube presence | No structured app features (replay, scoring, history) |  
| Puzzle pack monetization works | Web-based, not mobile-native |  
| Proves market for "Sudoku as craft" | Hardcore focus limits audience |  
| Celebrity constructor brand | No casual accessibility |  
  
| Opportunities | Threats |  
|---------------|---------|  
| Their community behavior proves demand for solve sharing and discussion | Their constructors are the brand — hard to compete on that axis |  
| — | Orbace can serve the same community need with better tools |  
| — | Their video-based solve sharing = latent demand for Su-Pu sharing |

| **Annotations** | Game annotations. Move comments. Personal notes. |

| **Collection** | Game archive with filters by date, opponent, opening, result. |  
  
**SWOT Analysis:**  
  
| Strengths | Weaknesses |  
|-----------|------------|  
| Game records are the core of the platform | Not in Sudoku category |  
| Every game is a replayable, shareable artifact | — |  
| Records create data moat (millions of games) | — |  
| Community built around game sharing and analysis | — |  
  
| Opportunities | Threats |  
|---------------|---------|  
| Their model proves Su-Pu works at scale | Could enter Sudoku market |

## 5. Deep Dive: Game Progress & Play History  
  
### 5.1 How Each Competitor Treats Player History  
  
| App | Philosophy | Player's Relationship to History |  
|-----|-----------|----------------------------------|  
| Sudoku.com | "Keep playing. Don't look back." | History doesn't matter. Just aggregate stats. |  
| Good Sudoku | "Here's what you've done recently." | History is a log. Glanceable, not explorable. |  
| Brainium | "We track everything. Here's the data." | History is a spreadsheet. Comprehensive, not warm. |  
| Andoku 3 | "All data is saved. Find it if you can." | History is a file system. Powerful, not designed. |  
| NYT Games | "Did you do today's puzzle?" | History is a streak counter. Habit, not journey. |  
| Cracking the Cryptic | "Complete the pack." | History is pack progress. Collection, not reflection. |  
  
### 5.2 What Everyone Does (Table Stakes)  
  
- Auto-save current game state: 100%  
- Resume from home screen: 100%  
- Basic stats (games played, best times): 90%  
- Streak tracking: 70%  
  
### 5.3 What Almost No One Does (Opportunity Gaps)

## 6. Deep Dive: Replay & Solve Recording  
  
### 6.1 The Current State  
  
**Replay does not exist in any mobile Sudoku app.**  
  
| App | Replay? | Closest Feature |  
|-----|---------|-----------------|  
| Sudoku.com | ❌ | None |  
| Good Sudoku | ❌ | Static completed board view |  
| Brainium | ❌ | Stats show what happened, not how |  
| Andoku 3 | ❌ | Move history for undo only |  
| Cracking the Cryptic | ❌ | Video export (browser-based, not mobile) |  
| NYT Games | ❌ | None |  
  
### 6.2 Why Replay is Absent  
  
1. **Data architecture**: Most apps store only final board state. Move-level recording requires forethought.  
2. **Perceived low demand**: Mass-market apps optimize for casual "complete and move on" behavior.  
3. **No competitive pressure**: Since no one has it, no one has been forced to build it.  
  
### 6.3 The Chess.com Proof of Concept  
  
Chess.com's game records (PGN format) demonstrate that recorded solves work at massive scale:  
  
- Millions of games recorded daily  
- Each game is replayable, shareable, analyzable  
- Players build collections of their games  
- Famous games are studied  
- Game records are the platform's core content  
  
**Sudoku has the same potential. No one has recognized it.**  
  
### 6.4 Orbace's Replay System (复盘)  
  
| Capability | Market Status | Orbace Implementation |  
|------------|--------------|----------------------|  
| Full move-by-move playback | None | 复盘 viewer with speed control |  
| Step list with technique labels | None | Each step labeled (NS, HS, NP, etc.) |  
| Player vs. optimal comparison | None | Side-by-side synchronized 对谱 |  
| Divergence detection | None | Auto-identify where paths split |  
| Replay without creating new attempt | None | Viewing is consequence-free |  
| Retry with version tracking | None | 重解 creates new Su-Pu with parent link |

## 7. Deep Dive: Scoring Systems  
  
### 7.1 How Competitors Score  
  
| App | Scoring Approach | Strengths | Weaknesses |  
|-----|-----------------|-----------|------------|  
| Sudoku.com | Time + difficulty tracked. No composite score. | Simple | Not a scoring system |  
| Good Sudoku | **No scoring.** Philosophical choice. | Avoids competition anxiety | No performance measurement |  
| Brainium | Points based on difficulty + time + errors | Most comprehensive in category | Opaque formula. Speed-dominant. |  
| Andoku 3 | Points based on difficulty + time | Customizable | Confusing defaults |  
| NYT Games | None. Solved/not solved. | Simple | No measurement |  
  
### 7.2 The Speed Problem  
  
All existing scoring systems are speed-dominant. This creates two problems:  
  
1. **Incentivizes rushing**: Players are rewarded for speed, not accuracy or elegance  
2. **Contradicts calm**: Timer pressure is the opposite of the calm experience many players want  
  
### 7.3 Orbace's Scoring Differentiation  
  
| Principle | Implementation |  
|-----------|---------------|  
| **Accuracy-dominant** | Errors and hints reduce a multiplier. Difficulty determines base. |  
| **Speed is optional** | Time bonus only when timer enabled. Capped at +10%. |  
| **Transparent** | Full breakdown shown: base, accuracy multiplier, each penalty, each bonus |  
| **Class-based** | 正谱 (official), 习谱 (practice), 重修谱 (retry), 旧谱 (legacy) |  
| **Deterministic** | Same puzzle + same performance = same score |  
| **Versioned** | Scoring version stored with every Su-Pu |  
| **Efficiency-aware** | Step count vs. optimal path tracked |
